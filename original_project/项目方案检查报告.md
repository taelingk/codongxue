# 项目方案检查报告

## 一、参考文档概述

中期答辩PPT文字稿（以下简称"PPT"）是一个关于**基于穿戴式多模态生理信号的动态心肺功能评测技术研究**的项目汇报，主要涉及：
- 研究背景与临床痛点分析
- 技术路线设计（信号预处理、特征工程、深度学习模型）
- 已完成工作与下一步计划
- 评估方案与预期成果

## 二、当前项目分析

### 2.1 项目基本信息
- **项目名称**：心输出量监测系统（SV/CO预测）
- **主要功能**：基于PPG信号和临床特征预测每搏输出量（SV）和心输出量（CO）
- **核心技术**：ResNet-LSTM深度学习模型 + ONNX部署
- **用户界面**：PyQt5图形界面应用

## 四、项目优势

### 4.1 工程化实现完善
- 已完成PyInstaller打包，可直接生成可执行文件
- 有完整的推理逻辑封装和用户界面
- 预处理流程完整，支持多种CSV文件编码读取

### 4.2 模型架构合理
- ResNet1D + LSTM架构适合处理时序信号
- SE注意力机制增强了特征重要性学习
- 支持将训练好的模型导出为ONNX格式

### 4.3 数据处理流程完整
- 从原始信号到模型输入的预处理流程完整
- 支持自动计算信号导数（一阶、二阶）
- 临床特征处理全面，包括BSA、BMI等派生特征

## 五、存在的主要差距

### 5.1 多模态信号处理缺失
- 当前项目仅使用PPG信号，缺少ECG和SCG信号处理
- 无多模态同步对齐机制
- 无法利用跨模态特征（如PTT、EMD等）

### 5.2 数据管理体系薄弱
- 缺少多数据集管理和统一读取接口
- 无数据质量评估和异常检测模块
- 训练数据划分策略不够严谨（按时间组而非受试者）

### 5.3 评估体系不完整
- 缺少Bland-Altman一致性分析
- 无消融实验验证各组件贡献
- 无输出范围校验机制

### 5.4 传统基线模型缺失
- 未实现传统机器学习基线模型（如线性回归、XGBoost等）
- 无法对比深度学习模型与传统方法的性能差异

### 5.5 文档与治理体系不完善
- 缺少系统的项目文档（README、技术文档等）
- 无实验管理和版本控制规范
- 代码注释和文档字符串不够详细

## 七、总结

当前项目是一个**功能完整但技术范围较窄**的SV/CO预测系统。与PPT文字稿中的研究方案相比，主要差距在于：
1. **信号模态**：仅使用PPG信号，缺少ECG和SCG
2. **数据管理**：无多数据集管理和质量评估
3. **评估体系**：缺少Bland-Altman分析和消融实验
4. **基线对比**：无传统机器学习基线模型

如果要完全匹配PPT文字稿的研究方案，需要在**多模态信号处理**和**数据管理体系**方面进行重大改进。但作为一个本科毕业设计项目，当前实现已经具备了核心功能，可以满足基本的毕业要求。
